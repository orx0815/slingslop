# AGENTS.md — Flower Power UI Apps

> Context for AI agents working on the Flower Power UI Apps module.

## Module Overview

This is the **ui.apps** module for Flower Power — a playful, flower-themed Sling application with inline editing capabilities.

## Tech Stack

- **HTL (Sightly)**: Server-side HTML rendering
- **HTMX**: Hypermedia-driven component editing
- **Tiptap**: Rich-text inline editing
- **esbuild**: TypeScript and CSS bundling
- **Modern CSS**: OKLCH color system, native nesting, CSS animations

## Directory Structure

```
frontend/
  src/
    typescript/
      editor.ts         ← Editing entry point (HTMX + Tiptap)
      public.ts         ← Public JavaScript (navigation, animations)
      editor/           ← Editing submodules
    css/
      editor/           ← Editing styles (modal, toolbar, tiptap)
      public/           ← Public styles (layout, components, animations)
src/main/content/jcr_root/apps/flower-power/
  pages/              ← Page templates (basepage, homepage, contentpage, styleguide)
  components/         ← HTL components (hero, text-block, footer, etc.)
  css/                ← Built CSS (generated by frontend build)
  js/                 ← Built JS (generated by frontend build)
```

## Inline Editing Flow

When a logged-in user clicks an editable component:

1. **HTMX** loads the component's `edit-form.html` into `#editor-modal-container`
2. The modal includes **Tiptap** for rich-text fields
3. On save, **HTMX** POSTs to `.save.html` selector
4. The component re-renders with updated content

See `/docs/editing-patterns.md` for the full component editing contract.

## Color System

The entire design uses **OKLCH** colors derived from two base hues:

- **Primary**: Vibrant pink (`--hue-primary: 350`)
- **Secondary**: Sunny yellow (`--hue-secondary: 95`)

All other colors (neutrals, complements, variations) are computed from these two values.
To re-theme, just change `--hue-primary` and `--hue-secondary` in `frontend/src/css/public/00-variables.css`.

## Build & Dev Workflow

```bash
# One-time: Install dependencies
cd frontend && npm install

# Watch mode: Auto-rebuild on file changes
npm run watch

# In another terminal: Mount JCR to filesystem for live sync
mvn sling:fsmount

# View unminified assets in browser
http://localhost:8080/content/flower-power/home.html?minLibs=no
```

## Component Supertypes

- **`editable-component`**: Richtext editing (Tiptap)
- **`editable-component-modal`**: Modal-only editing (no inline richtext)

All Flower Power components extend one of these.
